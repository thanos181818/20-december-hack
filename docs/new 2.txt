**Page: Product Page**

**Visible Tabs (Top Nav):**
- Products (active)
- Billing & Payments
- Terms & Offers
- Users & Contacts
- Reports
- User Profile (Username, My Profile, Sign Out)

**Product State Tabs:**
- New
- Confirmed
- Archived

**Product Fields (Left Section):**
- Product Name (text, required)
- Product Category (dropdown)
- Product Type (dropdown)
- Material (dropdown, optional)
- Color (multi-select)
- Current Stock (integer, required)

**Product Fields (Right Section):**
- Sales Price (decimal, required)
- Sales Tax (percentage/dropdown, optional)
- Purchase Price (decimal)
- Purchase Tax (percentage/dropdown)
- Published checkbox

**Additional Sections:**
- Product Images (multiple upload slots, + icon)

**Actions:**
- Save product
- Navigate between products (< > arrows)
- Switch product state tabs (New/Confirmed/Archived)
- Toggle published status
- Upload/remove images

**Validations:**
- Required: Product Name, Product Category, Sales Price, Current Stock
- Conditional: If published = true → must have Sales Price and at least one image
- Stock cannot be negative
- Only Confirmed products can be published

**Navigation Links:**
- Company Logo → Dashboard/Home
- Top Nav Tabs → respective modules
- User Profile → My Profile page or Sign Out → Login page
- Product State Tabs → filter products by status

**Page: Generate Coupon Modal/Page**

**Fields:**
- For selection dropdown: Customers / Vendors / Both
- Customers multi-select from contacts (hidden for anonymous flow)
- Quantity to generate (integer)
- Valid Until (date picker)

**Rules:**
- Anonymous Customers: Customer field hidden, user manually sets quantity
- Selected Customers: Customer field visible, quantity auto-calculated
- No Customer Selected: Coupons generated for ALL contacts, quantity = total contacts count

**Actions:**
- Generate button (creates coupon records)
- Cancel button (closes modal)

---

**Page: Coupons List Page**

**Table Columns:**
- Code
- Valid Until
- Program (offer name)
- Status (Used / Unused)
- Customer (assigned customer name)

**Example Rows:**
- 0440-a107-4a70 → Used → Pratik Patel
- 0442-f470-4614 → Unused → Raj Verma

**Rules:**
- Unused coupons can be used if within validity and offer active
- Used coupons locked, cannot be reused
- Page automatically filtered by offer

**Page: Sales Order Page**

**Controls:**
- New button (creates new SO, auto-generates SO number)
- Navigation arrows (< >) between orders

**Header Fields:**
- SO Number (auto-generated sequential, e.g., SO001)
- Customer Name (dropdown from Contacts & Users)
- Payment Term (dropdown from active payment terms)
- SO Date (defaults to today)

**Action Buttons:**
- Confirm (Draft → Confirmed, locks fields)
- Print (printable SO)
- Send (send to customer)
- Cancel (moves to Cancelled)
- Create Invoice button (visible after SO confirmed, creates/redirects to invoice)

**Line Items Table Columns:**
- Sr. No.
- Product (dropdown from Products table)
- Qty
- Unit Price (from product)
- Untaxed Amount
- Tax %
- Tax Amount
- Total

**Special Elements:**
- Discount line item (added when coupon applied, negative amount)
- Coupon Code button (only for Sale Order, opens modal)
- Status flow: Draft → Confirm → Cancelled

**Totals Section:**
- Untaxed Total
- Tax Total
- Grand Total (includes discount)

**Page: Customer Invoice Page**

**Controls:**
- New button (creates new invoice)
- Navigation arrows (< >) between invoices

**Header Fields:**
- Invoice Number (auto-generated sequential, e.g., INV/0001)
- Customer Name (dropdown from Contacts & Users)
- Payment Term (dropdown from active payment terms)
- Reference Sales Order Number (auto-filled if from SO)
- Invoice Date (defaults to today)
- Due Date (auto-calculated from payment term)

**Action Buttons:**
- Confirm (Draft → Confirmed)
- Print (printable invoice)
- Send (send to customer)
- Cancel (cancels invoice)
- Pay button (visible until invoice fully paid, opens payment page)
- Order button (visible if linked to sale order, redirects to SO)

**Line Items Table:** (same columns as Sales Order)
- Inherits from Sales Order if created from SO
- Discount line copied from SO

**Payment Tracking:**
- Paid On (date)
- Paid Amount
- Amount Due (auto-calculated)

**Status flow:** Draft → Confirm → Cancelled

**Page: Invoice Payment Page**

**Controls:**
- New button (creates new payment)
- Confirm/Print/Send/Cancel buttons

**Early Payment Discount Message:**
- Shows if payment term has early discount and payment date ≤ discount date

**Payment Form Fields:**
- Payment Type: Receive
- Partner Type: Customer
- Partner (auto-filled from invoice)
- Date (default today)
- Reference (default invoice number)
- Amount (auto-filled with Amount Due, editable for partial)

**Effects:**
- On confirm: reduces invoice Amount Due
- If Amount Due = 0: invoice marked Paid, Pay button disappears

**Navigation:**
- Button to navigate back to Invoice
- If invoice linked to SO: navigate to Sales Order
